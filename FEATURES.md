### 2. 🔄 智能故障切换系统

#### 工作原理
```
用户触发翻译
    ↓
尝试首选服务 (如 Google)
    ↓
失败？记录日志
    ↓
自动尝试下一个服务 (彩云)
    ↓
失败？继续记录
    ↓
自动尝试 DeepL → 百度 → 有道 → 腾讯
    ↓
成功返回结果 / 全部失败提示用户
```

#### 优先级顺序
1. Google 翻译 (默认)
2. 彩云小译
3. DeepL
4. 百度翻译
5. 有道翻译
6. 腾讯翻译

#### 日志输出示例
```
服务 google 翻译失败：Network error, 尝试其他服务...
正在尝试服务：caiyun
服务 caiyun 翻译失败：API token invalid
正在尝试服务：deepl
服务 deepl 翻译成功
```
