## 🔧 技术实现

### HTML 结构

每个服务配置块的结构：
```html
<div id="baiduConfig" class="service-config">
  <h3>📦 百度翻译</h3>
  
  <div class="form-group">
    <label>APP ID</label>
    <input type="text" id="baiduAppId" placeholder="请输入百度翻译 APP ID">
  </div>
  
  <div class="form-group">
    <label>密钥</label>
    <input type="password" id="baiduSecret" placeholder="请输入百度翻译密钥">
  </div>
  
  <!-- 新增测试按钮 -->
  <button class="btn btn-test test-service-btn" data-service="baidu">
    测试百度翻译
  </button>
  
  <p class="help-text">获取方式：https://...</p>
</div>
```

**关键点**:
- `class="test-service-btn"` - 标识这是测试按钮
- `data-service="baidu"` - 指定要测试的服务
- 按钮文本明确说明测试哪个服务

### JavaScript 逻辑

**事件绑定**:
```javascript
// 为所有测试按钮绑定事件
document.querySelectorAll('.test-service-btn').forEach(btn => {
  btn.addEventListener('click', async (e) => {
    const service = e.target.getAttribute('data-service');
    
    // 1. 检查配置是否填写
    // 2. 保存临时配置
    // 3. 执行测试
    // 4. 显示结果
  });
});
```

**测试流程**:
```
用户点击测试按钮
↓
读取 data-service 属性（如 "baidu"）
↓
检查对应配置是否填写完整
↓
如果未填写 → 提示用户
如果已填写 → 继续
↓
保存临时配置到 config.json
↓
调用 translate API 进行测试
↓
显示测试结果（成功/失败）
↓
恢复按钮状态
```

## 💡 智能特性

### 1. 自动检测配置

```javascript
// 自动检查对应服务的配置是否填写
if (service === 'baidu') {
  hasConfig = appId && secret;
} else if (service === 'deepl') {
  hasConfig = key;
}
// ... 依此类推
```

### 2. 临时配置保存

测试时会先保存临时配置：
```javascript
const tempConfig = {
  service: service,
  baidu: service === 'baidu' ? configFields : {},
  // ... 其他服务
};

await window.electronAPI.saveConfig(tempConfig);
```

**好处**:
- ✅ 确保测试时使用最新的配置
- ✅ 不会影响其他服务的配置
- ✅ 测试完成后可以恢复

### 3. 按钮状态管理

```javascript
// 测试中禁用按钮
e.target.disabled = true;
e.target.textContent = '测试中...';

// 测试完成恢复
e.target.disabled = false;
e.target.textContent = originalText;
```

**效果**:
- 防止重复点击
- 清晰的视觉反馈
- 用户体验更好

### 4. 详细的结果反馈

```javascript
if (result.success) {
  alert('✓ ' + service + ' 连接成功！\n翻译结果：' + result.result);
} else {
  alert('✗ ' + service + ' 连接失败：' + result.error);
}
```

**显示内容**:
- ✅ 服务名称
- ✅ 成功/失败状态
- ✅ 翻译结果或错误信息

### 常见问题排查

**问题 1: 点击测试按钮没反应**
```
检查：
1. 按钮是否有 data-service 属性
2. JavaScript 是否报错（F12 查看）
3. 配置是否填写完整
```

**问题 2: 测试一直失败**
```
检查：
1. API Key 是否正确
2. 网络连接是否正常
3. 查看错误信息具体内容
4. 检查服务是否可用
```

**问题 3: 测试成功但保存失败**
```
检查：
1. config.json 文件权限
2. 磁盘空间是否足够
3. 配置文件格式是否正确
```

## 📈 使用建议

### 最佳实践

1. **配置前先测试**
   - 申请到 API Key 后立即测试
   - 确认可用后再保存
   - 避免保存无效配置

2. **定期验证**
   - 每隔一段时间测试一次
   - 确认服务仍然可用
   - API 额度是否充足

3. **多服务备份**
   - 配置 2-3 个可用服务
   - 主力服务失败时快速切换
   - 提高翻译成功率

4. **记录关键信息**
   - API Key 申请日期
   - 免费额度多少
   - 何时到期
