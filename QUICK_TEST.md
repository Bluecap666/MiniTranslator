# 🚀 快速测试指南

## ⚡ 30 秒快速验证

### 步骤 1: 启动应用 (5 秒)

```bash
npm start
```

看到主窗口出现。

### 步骤 2: 打开配置 (5 秒)

点击主窗口右上角的 **⚙️** 图标。

看到紫色标题栏的配置窗口。

### 步骤 3: 测试关闭 (5 秒)

点击右上角的 **×** 按钮。

✅ 预期：窗口立即关闭。

### 步骤 4: 重新打开 (5 秒)

再次点击 **⚙️** 图标。

✅ 预期：窗口重新打开。

### 步骤 5: 测试最小化 (5 秒)

点击标题栏的 **−** 按钮。

✅ 预期：窗口最小化到任务栏。

从任务栏点击恢复窗口。

### 步骤 6: 测试取消 (5 秒)

点击底部的 **"取消"** 按钮。

✅ 预期：窗口立即关闭。

---

## ✅ 如果以上都通过

恭喜你！配置窗口的核心功能完全正常！🎉

## ❌ 如果有某步失败

请查看对应的错误情况：

### × 按钮无效
- 检查控制台是否有错误（F12）
- 确认看到 `[Config Window] 点击关闭按钮` 日志
- 如果没有，说明 preload.js 可能有问题

### − 按钮无效
- 同上检查日志
- 确认窗口没有设置为不可最小化

### 取消按钮无效
- 同上检查日志
- 确认按钮 ID 是 `cancelBtn`

### 窗口打不开
- 检查 ⚙️ 按钮是否绑定事件
- 查看 main.js 中 open-config-window 处理
- 确认 configWindow 变量正常

### 窗口无法拖动
- 确认标题栏有 `-webkit-app-region: drag`
- 检查 CSS 样式是否正确

## 🔍 详细调试方法

### 打开开发者工具

在配置窗口打开时按 **F12**。

### 查看控制台

切换到 **Console** 标签。

### 执行测试代码

在 Console 中输入：

```javascript
// 测试 API 是否存在
console.log('electronAPI:', window.electronAPI);
// 应该显示一个对象，包含 closeConfigWindow 等方法

// 测试按钮是否存在
console.log('closeBtn:', document.getElementById('closeBtn'));
// 应该显示 button 元素

// 手动触发关闭
window.electronAPI.closeConfigWindow();
// 窗口应该关闭
```
