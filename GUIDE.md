### 2️⃣ 智能故障切换

```
如果当前服务失败 → 自动尝试下一个服务
优先级：Google → 彩云 → DeepL → 百度 → 有道 → 腾讯
```

**示例场景：**
- 你在用 Google 翻译，但网络不好
- 系统自动尝试彩云小译
- 彩云也失败了
- 继续尝试 DeepL
- ✅ DeepL 成功返回结果！

## 🎯 快速上手

### 首次使用

1. **启动应用**
   ```bash
   npm start
   ```

2. **选择翻译服务**
   - 默认 Google 翻译（推荐）
   - 下拉框选择其他服务

3. **输入文本**
   - 在上半部分输入
   - 等待 1 秒自动翻译
   - 或按 Ctrl+Enter 立即翻译

4. **查看结果**
   - 下半部分显示翻译结果
   - 可以复制、编辑

### 配置 API（可选）

如果你想使用更多服务：

1. **点击设置图标 ⚙️**

2. **选择要配置的服务**
   - Google：无需配置 ✨
   - 彩云小译：填入 Token
   - DeepL：填入 API Key
   - 百度/有道/腾讯：填入对应凭证

3. **保存配置**
   - 点击"保存配置"按钮
   - 下次启动自动加载

### 使用快捷键

**场景 1：专注工作**
```
1. 正在写代码/写文档
2. 遇到不懂的英文
3. 按 Ctrl+Shift+X
4. 窗口出现，输入文本
5. 看到翻译后继续工作
6. 再按 Ctrl+Shift+X 隐藏窗口
```

**场景 2：频繁使用**
```
1. 设置默认服务为 Google
2. 窗口保持可见
3. 随时输入随时翻译
4. 不使用时拖到屏幕边缘
```

## 💡 最佳实践

### 推荐配置方案

#### 轻度用户
- **默认服务**: Google
- **备用服务**: 不配置
- **理由**: 完全免费，开箱即用

#### 专业用户
- **默认服务**: DeepL
- **备用服务**: Google + 彩云
- **理由**: DeepL 质量最高，其他作为备选

#### 开发用户
- **默认服务**: Google
- **备用服务**: 百度 + 有道 + 腾讯
- **理由**: 国内访问稳定，速度快

### 快捷键肌肉记忆

```
工作中：Ctrl+Shift+X → 输入 → 看结果 → Ctrl+Shift+X
学习时：保持窗口可见 → 随时输入 → 即时翻译
```

### 故障处理

**问题 1**: Google 翻译失败
```
解决：自动切换到其他服务
      或检查网络连接
```

**问题 2**: 所有服务都失败
```
解决：检查网络
      重启应用
      查看控制台日志
```

**问题 3**: 快捷键无效
```
解决：确保没有其他应用占用此快捷键
      重启应用
```

## ❓ 常见问题

**Q: 为什么需要这么多翻译服务？**
A: 不同服务各有优势，多服务保证稳定性和翻译质量。

**Q: 故障切换会影响速度吗？**
A: 轻微影响（约 0.5 秒），但大大提高了成功率。

**Q: 可以自定义快捷键吗？**
A: 当前版本固定为 Ctrl+Shift+X，后续版本支持自定义。

**Q: 窗口能最小化到多少？**
A: 300x200 像素，适合小屏幕使用。

**Q: 翻译历史会保存吗？**
A: 当前版本不保存，后续版本会添加此功能。
